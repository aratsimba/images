Question:
The pet shelter client's development team has set up an Amazon API Gateway endpoint to trigger their AWS Step Functions state machine. They want to ensure that they follow best practices for setting up the endpoint.

Which statement best describes why POST is the best HTTP verb to choose for the create-report endpoint?

Correct answer:
The action creates or initiates a new resource or process.

Distractors:
- The action is retrieving data idempotently.
- The action is updating existing resources in generating the report.
- The action removes resources from the system.

Response from Cline:

__Correct Answer: "The action creates or initiates a new resource or process."__

1. POST is the standard HTTP verb used when you want to create something new, like generating a fresh report.
2. The create-report endpoint starts a new process, making POST the appropriate choice according to REST API conventions.

__Distractor: "The action is retrieving data idempotently."__

1. This describes GET, which is used to read or fetch existing data without changing anything on the server.
2. Creating a report is not retrieval—it initiates a new process, so GET would be incorrect here.

__Distractor: "The action is updating existing resources in generating the report."__

1. This describes PUT or PATCH, which modify data that already exists rather than creating something entirely new.
2. The endpoint creates a new report each time; it does not update an existing report resource.

__Distractor: "The action removes resources from the system."__

1. This describes DELETE, which is used to remove or destroy existing resources from the server.
2. Generating a report adds new output rather than deleting anything, so DELETE does not apply here.

Response from Q:

Here are 2-sentence explanations for each option:

Correct Answer: "The action creates or initiates a new resource or process."
POST is designed for creating new things or starting processes on the server. When you trigger report generation, you're starting a new workflow execution, which is exactly what POST should do.

Distractor 1: "The action is retrieving data idempotently."
This describes GET requests, which fetch existing data without changing anything on the server. GET operations can be called multiple times safely and always return the same result.

Distractor 2: "The action is updating existing resources in generating the report."
This describes PUT or PATCH requests, which modify data that already exists. These verbs are for changing existing records, not starting new processes like report generation.

Distractor 3: "The action removes resources from the system."
This describes DELETE requests, which remove or destroy existing data from the server. DELETE is the opposite of creation - it eliminates resources rather than generating new ones.
